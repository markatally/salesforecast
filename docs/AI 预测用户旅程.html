01 / SELECT

选择销售预测场景

核心任务

选择与销售决策匹配的预测场景、对象粒度、覆盖范围和结果使用时间

关键动作

选择月内全国滚动、次月医院销量或未来六个月产品/规格销量，并筛选产品和区域

常见阻塞

将“30d”误解为未来三十天；未说明要服务的销售动作、产品或医院范围

平台期望

按业务语言展示场景、预测对象、时间口径和数据更新时间，避免依赖模型术语

02 / CHECK

确认结果与交付口径

核心任务

确认数据截止日、预测月份、全国或医院粒度、发布时间与可接受的误差范围

关键动作

查看任务状态和数据覆盖；确认月内结果是月末总量、次月结果含医院进货概率与销量

常见阻塞

数据流向延迟、医院/产品映射缺失，或销售口径与预测版本、实际值口径不一致

平台期望

展示运行进度、数据截止时间、质量告警、预计交付时间及可对比的历史预测版本

03 / ACT

消费预测并执行销售动作

核心任务

在销售窗口内用预测指导目标拆解、医院分层覆盖、资源安排和异常跟进

关键动作

查看预测值、医院进货可能性、实际与月末缺口；记录跟进、备货或资源调整动作

常见阻塞

结果靠人工 Excel 同步，缺少异常原因和动作建议；人工修订无法与原预测区分

平台期望

在预测门户提供筛选、版本化结果、历史对比、异常说明、订阅推送与人工覆盖留痕

04 / LEARN

回收实际并持续复盘

核心任务

回收实际销量和销售反馈，判断预测是否被采纳、是否改善了业务决策

关键动作

按产品、区域、医院和预测时点对比预测与实际；标注数据、市场或业务事件造成的偏差

常见阻塞

实际数据迟到，只看总体准确率；销售反馈未回流，无法区分模型偏差和业务变化

平台期望

提供预测—实际看板、WAPE/MAE 等切片评估、采纳率记录、反馈闭环和版本改进状态
